# RIP-0107 — AI Review Graph: Replacement of Layer 2B Reviewer Node

## 1. Purpose

This document proposes replacement of the current Layer 2B (L2B) independent reviewer node in the Meridian AI Review Graph. It defines the problem basis, replacement criteria, candidate evaluation, recommended replacement, implementation pathway, governance rules, and acceptance criteria for the change.

This proposal is subject to L2A (Claude) independent review, L2B review (see Section 9 — L2B gap note), Grok consolidation, and Final Approval by the Estate Principal.

**Recommended replacement: GitHub Copilot** (see Section 6).

## 6. Recommended Replacement: GitHub Copilot

**GitHub Copilot** is the recommended L2B replacement on the following basis:

- Satisfies all Hard and Soft criteria (Section 4)
- GitHub-native: OAuth-first, MCP-compatible, full read/write access — no new credential configuration required
- Preserves genuine vendor independence (Microsoft/GitHub, not Anthropic or OpenAI)
- No existing pipeline role — no echo chamber or independence risk
- Directly supports automation via the Meridian pipeline's existing GitHub MCP connector

**Independence note**: ChatGPT (L1) and GitHub Copilot (L2B) share GPT-4 class model lineage via Microsoft's OpenAI partnership. This is a residual independence concern the Estate Principal should acknowledge and accept. Vendor, infrastructure, system prompt, and dispatch pathway are entirely separate; shared model DNA is a theoretical rather than operational risk at this pipeline stage.

## 8. Implementation Pathway: GitHub Copilot as L2B

GitHub Copilot code review is **PR-native**. It reviews pull requests, not documents submitted via direct API. The Meridian pipeline requires the following adaptations.

### 8.1 Workflow Adaptation

Each governance document submitted for L2B review must be submitted as a pull request on a review branch. Copilot is requested as a reviewer on that PR. Its review appears as inline PR comments, which are extracted via the GitHub API for Grok consolidation.

**Per-version PR convention**: Each document version must create a fresh PR rather than updating an existing one. Copilot does not re-review automatically on push; re-review requires manual re-request via the UI.

### 8.2 Configuration Requirements

**Step 1 — Custom instructions file (mandatory)**
Create `.github/copilot-instructions.md` in the Meridian repository. This file embeds the Meridian L2B review system prompt — severity taxonomy, A1 standard requirements, RIP-003A alignment checks, finding format. Without this file, Copilot defaults to code-review behaviour on `.md` governance documents.

**Step 2 — Review effort level**
Set repository Copilot code review effort level to **Medium** in repository settings (Code & automation → Copilot → Code review → Review effort level). Medium routes to a higher-reasoning model for deeper analysis. Required for governance document quality.

**Step 3 — Review request**
Request Copilot review via the GitHub MCP `request_copilot_review` tool, or via the GitHub REST API with `copilot-pull-request-reviewer[bot]` as the requested reviewer. The Meridian pipeline's existing GitHub MCP connector supports this natively.

**Step 4 — Review extraction**
Read Copilot's review comments from the PR via the GitHub API for inclusion in the Grok consolidation package.

### 8.3 Known Constraints

| Constraint | Impact | Mitigation |
|---|---|---|
| PR-native only — no direct document API | Documents must be submitted as PRs | Establish per-version PR convention (Section 8.1) |
| No CI/CD pipeline integration | Cannot pipe results directly into n8n | Read PR review comments via GitHub API post-review |
| No automatic re-review on push | Each version requires fresh PR or manual re-request | Per-version PR convention (Section 8.1) |
| Code-review default behaviour | Governance documents reviewed as code without custom instructions | `.github/copilot-instructions.md` — mandatory (Step 1 above) |
| GPT-4 lineage overlap with L1 (ChatGPT) | Theoretical independence reduction | Accepted by Estate Principal — vendor, infrastructure, and dispatch pathway are independent |

### 8.4 Immediate Next Actions (Pre-Approval)

1. Draft `.github/copilot-instructions.md` with Meridian L2B system prompt
2. Confirm GitHub Copilot plan tier (Pro/Pro+/Max) is active on the repository
3. Run a validation review on one existing approved governance document
4. Confirm Copilot review output satisfies A1 finding format with custom instructions applied
5. Update `Registry/AI-Model-Registry.md` upon Estate Principal approval

## 9. L2B Reviewer Gap During Transition

- All consolidation artefacts produced during this period must be flagged "Incomplete/Fallback."
- Each Manual Adjudication instance requires explicit authorisation from the Estate Principal per RIP-003A §16.
- This document (RIP-0107) is itself subject to this fallback condition.

## 10. Governance Rules for Replacement Node

1. **Separation of decide and execute**: No single node may both decide (issue a review finding, recommendation, or consolidated output) and execute (commit content to the GitHub repository). These functions must be performed by different nodes or by the same node in clearly separated, auditable operations.
2. **Write authorisation**: Execution of GitHub write operations is limited to OAuth-authorised agents only.
3. **External verification**: No node may serve as the sole verifier of its own output.
4. **MCP preference**: MCP-compatible agents are preferred where equivalent alternatives exist.
5. **Registry maintenance**: All node assignments are recorded in `Registry/AI-Model-Registry.md` and updated upon approval of this document.

## 12. Rollback Procedure

If the replacement node fails acceptance criteria (Section 11):

1. Remove replacement node entry from AI Model Registry.
2. Revert n8n dispatch configuration to prior state.
3. Activate Fallback Policy per RIP-003A §16 or restore prior L2B node if available.
4. Raise a Decision Record documenting the failure, trigger conditions, and rollback action.
5. Notify the Estate Principal for disposition.
