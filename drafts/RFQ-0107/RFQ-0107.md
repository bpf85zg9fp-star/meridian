# RFQ-0107 — AI Review Graph: Replacement of Gemini Node

## 1. Context
The current AI review graph (per RIP-003A) utilizes a monolithic Gemini node for cross-model validation. This setup suffers from limited GitHub-native execution, insufficient MCP compatibility, legacy authorization, and constrained bidirectional repository mutation. These issues degrade determinism in review consensus.

## 2. Problem Statement
**Constraints:**
- Lack of MCP-native integration: Inhibits interoperability.
- Authorization Gaps: No standardized OAuth tool delegation.
- Write-loop Fragility: Weak GitHub write-side API integration.
- Orchestration Drift: High variance in outputs.

**Impact:**
- Inconsistent signals.
- Redundant cycles.
- Unpredictable PR merging.

## 3. Replacement Criteria
**Hard Requirements:**
- Full GitHub read/write via authenticated OAuth tokens.
- Mandatory OAuth for writes; MCP abstraction layer.
- Proven long-context reasoning and schema adherence.
- Strict determinism.

**Soft Requirements:**
- Multi-tool orchestration.
- Structured JSON/schema outputs.
- Advanced diff analysis.

## 4. Proposed Topology (Aligned with RIP-003A)
Multi-agent cascade for specialization:
**Claude (Reasoning Anchor)** → **ChatGPT (Orchestration Layer)** → **Copilot (Execution Authority)** → **Perplexity (Verification Layer)** → **Grok (Optional Signal / Determinism Layer)**

**Governance Enforcement (Section 5, RIP-003A):**
- Strict Separation of Powers: No node both decides logic and executes writes.
- All writes via MCP middleware + OAuth.
- Mandatory Perplexity sanity check on mutations.

## 5. Governance & Safety
- **Authorization**: OAuth tool delegation only.
- **Verification**: Perplexity low-confidence → PR Hold + Human intervention.
- **Standardization**: All comms via `schemas/` (A1 compliant).
- **Circuit Breaker**: Repetitive failures suspend autonomous writes.

## 6. Operational Modes
- **Light Review**: Claude + Copilot (speed).
- **Full Audit**: Complete cascade (sensitive areas).

## 7. Error Handling
- Fallbacks and circuit breakers as above.
- Logging to event store per governance_kernel.py.

## 8. Recommendation & Next Steps
Approve replacement of Gemini node with proposed topology. 
**Immediate Actions:**
1. Develop MCP middleware (reference .github/workflows/*).
2. Update AI Model Registry in governance/registry/.
3. Layer 2 review (Claude/ChatGPT).
4. Human Authority (Kevin) gate before merge.

**Implementation Notes**: Latency mitigation via parallel light paths; test in agent-arbitration.yml. Feedback on middleware welcome.
